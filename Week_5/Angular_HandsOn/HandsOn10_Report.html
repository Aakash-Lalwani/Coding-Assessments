Task 2: Service & NgRx Store Testing

CourseService tests with HttpTestingController and MockStore testing

Service Tests with HTTP Mock

  • HttpClientTestingModule: Mocks HTTP backend for testing without real server
  • httpMock.expectOne(url): Asserts a single request to expected URL was made
  • req.flush(data): Simulates HTTP response with mock data
  • httpMock.verify(): Ensures no outstanding requests remain (catches unintended calls)

NgRx Store Testing with MockStore

  • provideMockStore({ initialState }): Provides controllable mock store
  • store.setState(): Simulates state changes without real reducers/effects
  • MockStore: Allows testing NgRx-connected components in isolation
