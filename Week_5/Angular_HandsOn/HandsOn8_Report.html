Task 2: RxJS Operators

  • map: Transforms response - filters courses with credits > 0
  • tap: Side effects (logging) without modifying stream data
  • retry(2): Automatically retries failed requests up to 2 times
  • catchError: Catches errors and returns user-friendly error message
  • switchMap: Chains HTTP calls, cancels previous inner observable if new request arrives

Task 3: HTTP Interceptors

Auth interceptor, loading interceptor, and error handler interceptor

Three Interceptors Registered

InterceptorPurposeOrder
authInterceptorAdds Authorization: Bearer header to all requests1st
errorHandlerInterceptorHandles 401 (redirect) and 500 (alert) globally2nd
loadingInterceptorShows/hides global loading spinner via BehaviorSubject3rd

Execution Order: Request goes through 1 → 2 → 3. Response travels back 3 → 2 → 1.

Expected Outcome

DevTools shows Authorization header on all API requests. Simulating 401 navigates to home. Spinner appears on every HTTP call and disappears when complete (using finalize). Course list loads from JSON Server. CRUD operations persist to db.json.
