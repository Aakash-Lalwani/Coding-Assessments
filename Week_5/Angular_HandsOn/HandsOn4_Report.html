Task 2: Validation and Error Messages

Validation Approach

  • Template reference variables: #nameCtrl='ngModel' for accessing control state
  • Error display timing: Show errors only when control is touched (not dirty)
  • CSS classes: .ng-invalid.ng-touched { border-color: red } / .ng-valid.ng-touched { border-color: green }
  • Conditional error messages: Specific messages for each validation error type

Features

  • Submit button disabled when form is invalid
  • Success message appears after successful submission
  • Reset button clears all fields and validation states via enrollForm.resetForm()

Expected Outcome

Invalid fields show red borders and specific error messages after being touched. Valid fields show green borders. Successful submission shows the success message. Console logs form value object with all field names as keys on submit.
