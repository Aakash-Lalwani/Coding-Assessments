Task 1: CourseService (Singleton)

CourseService with providedIn: 'root', injected into multiple components

Service Architecture

  • CourseService: Manages course data with getCourses(), getCourseById(), addCourse()
  • providedIn: 'root': Creates a single shared instance across the entire application
  • TypeScript Interface: Course model ensures compile-time type checking

Multiple components (CourseList, Home, CourseSummaryWidget) inject the same CourseService instance. Adding a course in one component updates the count in all others - confirming singleton behavior.

Task 2: EnrollmentService & Hierarchical DI

Component-level provider creating separate service instance per component

Key Patterns

  • Service-to-Service injection: EnrollmentService injects CourseService to resolve course IDs
  • Component-level providers: providers: [NotificationService] creates separate instance scoped to that component
  • Hierarchical DI: Angular searches injectors up the tree - Component → Parent → Module → Root

Expected Outcome

Enroll button toggles to Unenroll after clicking. Enrolled courses appear in profile page. Course count in summary widget matches the service data. Component-level provider creates isolated NotificationService instance.
