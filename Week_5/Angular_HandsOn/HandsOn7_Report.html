Task 2: Lazy Loading & Route Guards

AuthGuard (CanActivate) and UnsavedChangesGuard (CanDeactivate)

Guards Implemented

  • AuthGuard (CanActivate): Checks isLoggedIn, redirects to home if not authenticated
  • UnsavedChangesGuard (CanDeactivate): Shows confirm dialog if reactive form has unsaved changes

Lazy Loading

Enrollment feature module is only downloaded when user navigates to /enroll. Verified via Chrome DevTools Network tab - separate JS chunk loaded on first navigation.

Expected Outcome

Network tab shows separate chunk loaded on first /enroll navigation. Navigating to /profile without login redirects to home. Dirty form shows confirmation dialog on navigation away. /courses/2 displays correct course detail.
