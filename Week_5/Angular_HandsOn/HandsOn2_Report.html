Task 1: All Four Binding Types

Implementation of all 4 Angular binding types in HomeComponent
Binding TypeSyntaxDirectionExample
Interpolation{{ expression }}Component → DOM<h1>{{ portalName }}</h1>
Property Binding[property]="value"Component → DOM<button [disabled]="!isActive">
Event Binding(event)="handler()"DOM → Component<button (click)="onClick()">
Two-Way Binding[(ngModel)]="prop"DOM ↔ Component<input [(ngModel)]="searchTerm">

Expected Outcome

  • Typing in search box updates "Searching for:" text in real time
  • Clicking "Enroll Now" shows the message
  • Button disabled state changes based on isPortalActive

Task 2: Lifecycle Hooks

ngOnInit, ngOnDestroy in HomeComponent; ngOnChanges in CourseCardComponent

ngOnInit fires once after inputs are first set - use for data fetching.

ngOnDestroy fires before component destruction - use for cleanup (unsubscribe, clear timers).

ngOnChanges fires when @Input properties change - log previous and current values.

Task 3: @Input and @Output Communication

Parent-to-child data flow with @Input, child-to-parent events with @Output

Expected Outcome

Clicking Enroll on any card logs the course ID to console and shows selected ID below the list. Data flows down via @Input, events bubble up via @Output.
