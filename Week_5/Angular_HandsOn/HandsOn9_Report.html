Task 2: Effects & Enrollment State

CourseEffects with switchMap for HTTP, cross-slice enrollment selectors

Effects Flow

  1. Component dispatches loadCourses()
  2. Effect listens via ofType(loadCourses)
  3. Effect calls courseService.getCourses() via switchMap
  4. On success: dispatches loadCoursesSuccess({ courses })
  5. On error: dispatches loadCoursesFailure({ error })
  6. Reducer updates state based on action type
  7. Selector emits new value, component re-renders via async pipe

Cross-Slice Selectors

selectEnrolledCourses combines course state + enrollment state to derive full Course objects for enrolled IDs without duplicating state.

Expected Outcome

Redux DevTools shows the complete action → effect → success flow. Enrollment state updates correctly. The async pipe renders enrolled/unenrolled state per card. State tree shows courses, loading, and error properties.
